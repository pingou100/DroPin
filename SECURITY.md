# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in DroPin, please report it by:

1. **Email:** [Your email or create a GitHub security advisory]
2. **GitHub Issues:** For non-sensitive issues, you can also open a public issue

## What to Report

- Security vulnerabilities in the code
- Potential data leaks
- XSS or injection vulnerabilities
- Authentication/authorization issues

## What NOT to Report

- Issues with your own API keys (these are your responsibility)
- Browser compatibility bugs (use regular issues)
- Feature requests (use regular issues or discussions)

Thank you for helping keep DroPin secure!
